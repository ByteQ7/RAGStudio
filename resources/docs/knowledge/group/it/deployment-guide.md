# 部署与运维手册

## 一、环境说明

| 环境 | 用途 | 部署方式 |
|------|------|---------|
| dev | 开发自测 | Docker Compose |
| test | 集成测试 | Docker Compose |
| staging | 预发布验证 | Kubernetes |
| production | 生产环境 | Kubernetes |

## 二、部署流程

### 标准发布流程
```
代码合并到 main → CI/CD 自动构建 → 镜像推送至 Harbor → Staging 部署验证 → 生产发布
```

### 版本号规范
- 格式：`v主版本.次版本.修订号`，如 `v2.3.1`。
- 主版本：重大架构变更。
- 次版本：功能迭代。
- 修订号：Bug 修复。

### 发布审批
- Staging 环境：开发负责人确认即可。
- 生产环境：需经测试负责人、技术负责人两级审批。
- 紧急发布：可简化审批流程，但需在发布后 24 小时内补全审批。

## 三、监控告警

### 监控指标

| 指标 | 告警阈值 | 处理时限 |
|------|---------|---------|
| CPU 使用率 | > 80% 持续 5 分钟 | 30 分钟 |
| 内存使用率 | > 85% 持续 5 分钟 | 30 分钟 |
| 磁盘使用率 | > 90% | 1 小时 |
| API 响应时间 P99 | > 3 秒 | 15 分钟 |
| 错误率 | > 1% 持续 5 分钟 | 10 分钟 |
| 服务可用性 | < 99.9% | 立即响应 |

### 告警方式
- 一级告警（紧急）：电话 + 短信 + 企业微信
- 二级告警（重要）：短信 + 企业微信
- 三级告警（一般）：企业微信

## 四、日志管理

- 应用日志统一输出到 `stdout`，由容器运行时收集。
- 日志保留 30 天，归档日志保留 6 个月。
- 严禁打印敏感信息（密码、身份证号、银行卡号）。
- 异常日志需包含完整的 stack trace 和请求上下文。
