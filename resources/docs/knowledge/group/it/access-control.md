# 信息系统权限管理制度

## 一、权限管理原则

1. **最小权限原则**：用户仅拥有完成工作所需的最小权限集。
2. **职责分离原则**：敏感操作需多人审批或双人操作（如财务付款、用户删除）。
3. **定期审计原则**：权限分配情况每季度审计一次。
4. **申请审批原则**：所有权限变更需通过 OA 流程申请和审批。

## 二、账号生命周期管理

### 账号创建
- 新员工入职当日由 IT 部门根据 HR 入职通知创建账号。
- 账号权限模板根据岗位确定。
- 默认开通：OA、邮箱、企业微信。

### 权限变更
- 岗位变动时，由调入部门负责人提交权限变更申请。
- 调出部门需同时提交原岗位权限回收申请。

### 账号禁用
- 员工离职当日由 IT 部门禁用全部系统账号。
- 长期休假（超过 30 天）需申请账号临时禁用。
- 账号禁用后保留 90 天，确认无需要后删除。

## 三、敏感权限管理

| 敏感权限 | 申请条件 | 审批流程 |
|---------|---------|---------|
| 数据库直接查询 | 开发/运维人员 | 技术负责人 → IT 总监 |
| 生产服务器 SSH | 运维人员 | 运维负责人 → IT 总监 |
| 管理员后台 | 系统管理员 | 部门负责人 → 分管领导 |
| 批量数据导出 | 指定人员 | 数据安全委员会审批 |
| 配置修改 | 系统管理员 | 变更管理流程 |

## 四、密码策略

- 最小长度 10 位，包含大写、小写、数字、特殊字符。
- 有效期 90 天，不可与前 3 次密码重复。
- 连续 5 次登录失败锁定 30 分钟。
- 首次登录强制修改初始密码。
- 每季度进行弱密码扫描，发现弱密码强制修改。
