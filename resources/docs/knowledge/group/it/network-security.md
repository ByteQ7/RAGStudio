# 网络安全管理制度

## 一、总则

为保障公司信息系统和网络安全，防止数据泄露和网络攻击，特制定本制度。

## 二、账号与密码管理

1. 所有系统账号实行实名制管理，严禁共享账号。
2. 密码复杂度要求：至少 8 位，包含大小写字母、数字和特殊字符。
3. 密码每 90 天必须更换一次，不得与前 5 次重复。
4. 连续 5 次登录失败将锁定账号 30 分钟。
5. 离职员工账号需在办理离职手续当日由 IT 部门禁用。

## 三、网络使用规范

1. 公司网络分为内网和外网，内网用于处理敏感数据。
2. 严禁私自搭建无线热点或连接未经授权的 Wi-Fi。
3. 严禁使用公司网络进行与工作无关的大流量下载（视频、游戏等）。
4. 外部访客需使用访客 Wi-Fi，由接待人申请开通。

## 四、数据安全

### 数据分级

| 级别 | 定义 | 示例 |
|------|------|------|
| 绝密 | 泄露会对公司造成重大损失 | 核心算法、客户数据库、财务数据 |
| 机密 | 泄露会对公司造成较大损失 | 合同、薪酬数据、战略规划 |
| 内部 | 限于公司内部使用 | 内部制度、会议纪要 |
| 公开 | 可对外公开的信息 | 宣传资料、招聘信息 |

### 数据加密
- 绝密和机密数据传输必须使用加密通道（HTTPS、SFTP、VPN）。
- 笔记本电脑硬盘必须启用全盘加密（BitLocker/FileVault）。
- 移动存储设备（U 盘、移动硬盘）需经 IT 部审批后方可使用。

## 五、邮件安全

1. 公司邮箱仅限工作使用，不得用于注册个人网站和服务。
2. 不打开来源不明的邮件附件和链接。
3. 发现钓鱼邮件立即报告 IT 服务台。
4. 发送含敏感信息的邮件务必使用加密功能。

## 六、安全事件应急响应

1. **发现阶段**：发现安全事件后立即报告 IT 服务台。
2. **处置阶段**：IT 部门 30 分钟内启动应急响应流程。
3. **恢复阶段**：修复漏洞，恢复系统正常运行。
4. **复盘阶段**：5 个工作日内完成事件复盘报告。
