# 数据库管理规范

## 一、数据库选型

| 场景 | 推荐数据库 | 说明 |
|------|-----------|------|
| 业务数据 | PostgreSQL 14+ | 主业务数据库 |
| 缓存 | Redis 6+ | 热点数据缓存、分布式锁 |
| 向量存储 | pgvector | 向量相似度检索 |
| 全文检索 | PostgreSQL tsvector | 关键词搜索 |
| 消息队列 | RocketMQ 5.2 | 异步消息、事件驱动 |

## 二、表结构设计

### 命名规范
- 业务表：`t_` + 业务名称，如 `t_user`、`t_order`。
- 关联表：`t_` + 实体1 + `_` + 实体2，如 `t_user_role`。
- 字段名使用小写蛇形命名，如 `create_time`。

### 必须字段
每张表必须包含以下字段：
```sql
id          VARCHAR(64) PRIMARY KEY,       -- 雪花算法 ID
create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
deleted     SMALLINT DEFAULT 0             -- 逻辑删除标记
```

### 索引规范
- 查询频繁的字段必须建索引。
- 联合索引遵循最左前缀原则。
- 索引名格式：`idx_` + 表名缩写 + `_` + 字段名。
- 避免在 WHERE 条件中对索引字段使用函数。

## 三、SQL 编写规范

- 严禁使用 `SELECT *`，必须明确列出字段。
- 大批量数据操作使用批量处理（batch），单次不超过 1000 条。
- 复杂查询优先使用应用层组装，而非数据库函数。
- 事务中严禁执行远程 RPC 调用。
- 长事务控制在 5 秒以内。

## 四、备份策略

| 备份类型 | 频率 | 保留时间 |
|---------|------|---------|
| 全量备份 | 每日凌晨 2:00 | 30 天 |
| WAL 归档 | 实时 | 7 天 |
| 逻辑备份 | 每周日 | 3 个月 |
